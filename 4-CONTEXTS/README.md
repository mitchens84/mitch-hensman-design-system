# Tier 4 — Contexts

Per-surface application layer. Each file describes how Foundations + Tokens + Patterns apply to a specific output surface. This tier is extensible — add new context files when new surfaces emerge (e.g., a future podcast, video, newsletter).

## Files

- `WEBSITE.md` — mitch-hensman.com; points to website repo as implementation
- `DOCUMENTS.md` — MD/PDF/DOCX/EPUB; absorbs STD-STYLE_GUIDE--DOCUMENT_PRESENTATION
- `PRESENTATIONS.md` — decks, slide masters
- `SOCIAL.md` — Twitter/X, LinkedIn, Telegram, WhatsApp
- `EMAIL.md` — HTML email templates, plain-text, email signatures
- `AI-CONVERSATION.md` — Claude chat; canonical source for CLAUDE.md §0.3
- `REPORTS.md` — daily briefings, weekly updates, meeting minutes
- `CLIENT-DELIVERABLES.md` — commercial register (Sitruna, Connexus)
- `TECHNICAL.md` — CLAUDE.md files, STD docs, code comments, README

## Per-goal overrides

`GOAL-OVERRIDES/` holds LBS- or goal-specific voice and visual adaptations that compose with foundation voice. Example: Sitruna commercial work may override voice rules with a client-formal register declared in `GOAL-OVERRIDES/7A-SITRUNA.md`.
